How do they work?

Transmission Control Protocol - the basic communication language of the internet

WebSocket Security

WSS (WebSockets over SSL/TLS)

  1. Client asks Server for authorization ticket
  2. Server creates the ticket (username/password combination, ip address, etc.)
  3. Server stores the ticket and returns it to Client
  4. Client opens WebSocket, sending this ticket as part of the "handshake" event
  5. Server can repeatedly check ticket for authenticity, expiration time, etc.
